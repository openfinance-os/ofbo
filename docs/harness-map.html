How this was built

A bank back office, built by an AI software factory — with humans only where it counts.

OFBO is 135 shipped features across 740 files: reconciliation, customer care, risk, analytics — regulated UAE Open Finance. Almost none of it came out of a meeting. An autonomous agent harness runs the full lifecycle — discover, define, develop, deliver — and stops for a human at exactly three decisions.

Below: how the loop works, what it replaces, the cast of skills · agents · hooks · scripts that make it safe, and the three points where a person stays accountable. Click any tile for the detail.

The model

One disciplined loop: a regulated Double Diamond

The harness widens then narrows twice — first on the problem (Discover ▸ Define), then on the solution (Develop ▸ Deliver). Each phase is an agent skill; deterministic gates and independent AI reviewers sit between them; humans hold the joints.

◇ waist gate · a human accepts the problem ✋ merge gate · a human merges every PR ✋ prod gate · a human approves release

What it replaces

Before & after: from a calendar full of people to three decisions

The conventional way a regulated bank ships one feature is a chain of human ceremonies, each with its own meeting, attendees, and hand-authored artifact. The harness keeps the rigor — the same checks still happen — but runs them as code and agents, and calls a human only when accountability genuinely requires one.

The cast & why

What makes it safe enough for a bank

Autonomy without guardrails is a non-starter in a regulated shop. So the harness is layered: agents do the open-ended work, but every consequential step is fenced by something deterministic. Each layer exists for a specific reason — click any tile to see it.

Human-in-the-loop

Three decisions a person still owns

The agent proposes; a human disposes. Everywhere else the machine is faster and more consistent — but these three carry regulatory accountability, so they never run on AI judgement alone.

Everything between these points — discovery, evidence, design, code, tests, contract checks, lineage, the pull request itself — is produced and verified by the harness. The human reads, questions, and decides. That is the whole design: machines do the work; people own the calls that matter.
